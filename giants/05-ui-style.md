# 05 · UI 視覺規範 UI Style Guide

## 設計核心原則

> **「環境即 UI。」**

所有遊戲狀態透過場景物件、角色動作、光影變化傳達。**不顯示任何傳統 HUD 元件**，保持畫面乾淨與沉浸感。

---

## 色彩規範

### 主色盤

| 色彩名稱 | Hex 值 | 用途 |
|----------|--------|------|
| **Sand（沙）** | `#D4C5A9` | 主要地面材質、UI 背景 |
| **Moss（苔）** | `#7A9E7E` | 植被、生命相關視覺 |
| **Sky（天）** | `#4A6FA5` | 天空色調、傳送門光效 |
| **Stone（石）** | `#2D2D2D` | 巨人建築、陰影區域 |
| **Gold（金）** | `#E8C87A` | 弱點標記、收集品、重要互動點 |
| **Danger（危）** | `#C04C3A` | Boss 攻擊預警、危險區域 |

### 色彩使用規則

- **金色（Gold）** 是玩家最需要注意的顏色：弱點、可互動物件、收集品都使用此色
- **危險色（Danger）** 僅在 Boss 攻擊前 0.5 秒的地面預警標記使用，全遊戲唯一的 UI 層顏色
- **其他色彩** 均為環境色，不作為 UI 信號使用

## 字體規範

| 用途 | 字體 | 理由 |
|------|------|------|
| **標題 / 遊戲 Logo** | Cinzel | 石刻質感，呼應古老巨人文明 |
| **系統文字（選單等）** | Noto Sans TC | 中英混排清晰，跨平台兼容 |
| **資料型文字（收集品資訊）** | Share Tech Mono | 等寬碼字感，製造「發現文明遺物」的感覺 |

## 選單設計

### 主選單

**設計方向：** 用場景本身作為主選單背景——玩家第一次啟動遊戲時，主選單顯示在廣原的日出場景中，菜單項目以透明疊加方式出現在場景上方。

| 選單項目 | 中文 | 排版位置 |
|----------|------|----------|
| New Game | 開始旅程 | 畫面中央偏下 |
| Continue | 繼續 | 開始旅程下方 |
| Settings | 設定 | 頁面右下角（低視覺優先度） |

> **設計決策：** 「設定」刻意放在低視覺優先位置，因為本作鼓勵玩家「直接開始探索」而非在選單停留。

### 暫停選單

暫停時畫面**不完全黑暗**，而是輕微去飽和度（降至 40%）並保留場景可見性，強調「你只是暫停了，世界還在那裡」。

## 音效與 UI 的整合

本作沒有傳統的「UI 音效」（例如按鈕點擊聲），所有聲音反饋都來自場景音效層：

| 狀態 | 音效反饋 |
|------|----------|
| 進入可互動範圍 | 風聲輕微改變頻率 |
| 完成互動 | 石頭摩擦聲 / 金屬共鳴聲 |
| 受傷 | 旅人呼吸急促 + 低沉打擊音 |
| 收集彩蛋 | 短暫輕快音符（2 音節，不干擾環境音） |
| Boss 警戒 | 大地低頻震動 |

## 版本迭代紀錄

| 版本 | 主要 UI 變更 | 原因 |
|------|-------------|------|
| v1.0 | 有血量條（畫面左上） | 初始設計 |
| v1.2 | 移除血量條，改為斗篷狀態 | Playtest：血量條破壞沉浸感 |
| v1.5 | 移除所有地圖元素 | 玩家反映「地圖讓探索失去意義」 |
| v2.0 | 加入受傷音效補償 | 無 HUD 後玩家不知道自己受傷 |
| v2.1 | 統一互動點視覺為金色 | 降低學習曲線，提高一致性 |
