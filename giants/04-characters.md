# 04 · 角色設計 Character Design

## 設計原則

本作所有角色設計遵循同一個核心原則：**角色的外觀與行為必須服務於世界觀與情感目標，而非追求「有趣」或「酷炫」。**

---

## 旅人（Player Character）

### 設計概述

| 屬性 | 設定 |
|------|------|
| **外觀** | 全身斗篷，面部完全遮蔽 |
| **性別 / 年齡** | 刻意不明確 |
| **體型** | 渺小——相對於場景環境刻意縮小比例 |
| **能力成長** | 體力槽隨旅程進度擴展 |

### 設計意圖

**為什麼遮蔽面部？**

讓玩家投射自我。當角色沒有明確的面孔時，任何玩家都更容易認同「這就是我」——不受性別、年齡、種族的限制。這是 Journey（2012）同樣使用的設計策略。

**為什麼刻意縮小體型？**

場景尺度是這個遊戲最重要的視覺語言。旅人的渺小讓巨人的巨大有了對比基準，同時強化「廣闊世界中只有你一個人」的孤獨感——如果角色太大，世界看起來就不夠廣闊了。

### 能力數值設計

| 數值 | 初始值 | 最終值 | 成長方式 |
|------|--------|--------|----------|
| 移動速度 | 8 m/s | 10 m/s | 線性成長（每完成一個主要區域 +0.5） |
| 滑行最大速度 | 20 m/s | 30 m/s | 隨關卡解鎖 |
| 體力槽（攀爬用） | 60% | 100% | 收集特定遺物後擴展 |
| 跳躍高度 | 基礎值 | 基礎值 × 1.3 | 特定彈跳板後臨時提升 |

**成長弧線設計意圖：**
旅人的能力成長不靠「打怪升級」，而是靠「探索這個世界」——每完成一個主要區域，旅人的移動能力就輕微提升，暗示他在這段旅程中真實地成長了。

### 斗篷狀態系統（敘事型狀態回饋）

| 狀態 | 斗篷視覺 | 觸發條件 |
|------|----------|----------|
| 健康 | 完整、飄逸 | 正常狀態 |
| 輕傷 | 邊緣略有磨損，顏色稍暗 | 受到一次傷害 |
| 重傷 | 多處撕裂，顏色明顯暗沉 | 受到兩次傷害 |
| 收集增長 | 逐漸增加裝飾細節 | 每收集 10 個彩蛋 |

## 沉睡巨人 · Boss A

### 設計概述

| 屬性 | 設定 |
|------|------|
| **狀態** | 半覺醒 |
| **體積** | 旅人的 15–20 倍身高 |
| **移動速度** | 極慢（0.5 m/s），但範圍極大 |
| **攻擊方式** | 踩踏、揮手掃除 |
| **弱點位置** | 背部上方，金色符文標記 |

### 設計意圖

**為什麼 Boss 在戰後不死？**

巨人在 Boss 戰後只是再次沉眠，而非死亡。這個設計決策來自世界觀核心：巨人不是「敵人」，只是存在於不同時間尺度的生命體。旅人不是來消滅巨人的——他只是需要「讓沉眠的巨人繼續沉眠」才能通過。

這個設計傳達的情感是：**敬畏，而非勝利感**。

### Boss 行為模式

```
Phase 1（初始）：
  → 緩慢移動，偶爾隨機踩踏
  → 玩家可安全接近、觀察

Phase 2（玩家首次攀附）：
  → 巨人察覺到異物，移動速度稍微加快
  → 開始甩手動作（玩家需抓緊）

Phase 3（首次弱點觸發後）：
  → 巨人開始更頻繁地甩手
  → 短暫倒地（5 秒窗口讓玩家攀爬至弱點）

結局（三次弱點觸發後）：
  → 巨人緩慢坐下，符文熄滅
  → 旅人站在巨人肩膀上，鏡頭拉遠展示二者體型對比
  → 過場：巨人手緩緩落地，旅人跳下繼續前進
```

## 靜態巨人（Non-Encounter）

分散在各個場景的沉眠巨人，玩家可以攀爬探索，但不會觸發 Boss 戰。

**設計功能：**
- 作為地標，幫助玩家在無地圖的世界中定向
- 提供攀爬樂趣，讓玩家練習攀爬操作
- 部分靜態巨人身上藏有彩蛋或世界觀碎片
- 讓玩家「習慣」與巨人共存，為後續 Boss 遭遇製造對比（「我以為巨人都是安全的...」）
